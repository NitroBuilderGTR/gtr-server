GTR_Server: Open Alpha v4 - Patch Notes

Notes: Mainly new features, and huge map changes

Changes:

- Updated Zeon_Launcher, to run the new version of GTR_Server (GTR_Server: Open Alpha v4)

- Added new building. Possibly part of storyline? Lore? Who knows.

- Added The Race Track. Can be found at the right of the spawn building. (FIRST EVER MINIGAME FOR GTR_Server)

- Added Race Lights for the Race Track

- Performance changes. Not sure if it runs better, but I tried to optimise the performance. You can still change your performance settings if GTR_Server doesnt run well for you.

- Added Player-Only and Server-Host-Only Cars.

- Added Player Beds

- Added Player Toilets

- Added Baths

- Added Sinks

- Added Bathrooms to Player Houses

Bugs

Major Bugs

1) The "Admin Control Panel" doesn't work. This feature isn't finished yet.

Solution: Just don't try to use it - it doesn't work lol

2) Emergency Respawn Button doesn't work

Solution: If you need to respawn press H. If it doesnt work, please restart your application.

3) Stuttering in non-server owner's car

Solution: This is not going to be a easy fix (might take a extremely long time to fix, and there is no guarantee that this gets fixed). So you could stay on the old version (if you have it downloaded), or just wait until it's fixed. The suspected reason on why it stutters is the multiplayer replication being a bit weird for the player cars.

4) FPS / Performance Lag

Solution: Go to The In - Game Settings. You can then turn shadows to the lowest setting. It may look worse, but it gives more FPS. This is recommended for Lower - End PCs / Devices.

Bug Fixes (Insane progress)

1) Fixed when running into car, and getting despawned

Solution: The issue was something to do with the server (person who hosted the server) having more "access" or having "priority" to the car, compareed to a player who isn't classed as the server (someone who joined the server instead). This was an easy fix. Now there are 2 types of cars - some for the server owner (The Admin Car), and some for the players (The Player Car).

2) Chairs being buggy

Solution: Chairs were quite easy to fix - turned out the replication isn't easy to do lol

3) The Performance is not great near loads of items and/or vehicles

Solution: I have been trying to find a optimisation solution for this. It turned out there was a "null" AI controller that possessed the cars. This was potentially one of the reasons why the performance would be tanked everytime you load in. The performance issue still isn't fixed, but I think the issue has been temporarily solved. The performance still isn't great, but I think it has been made better by around 1% to 10% after removing the AI Controllers from the vehicles.
